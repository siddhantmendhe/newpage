Project overview

The problem:

Restaurant customers were not able to track their food orders in real-time.





The goal:

Make an order tracking process easier, so customers don’t have to call restaurant staff for their order status.

Delivery address changing

For each order, the delivery person can know where to deliver by seeing the delivery address on an app.

Relaible way of tracking orders

Using a mobile app can solve this issue.

Feedback giving

The feedback-giving process is long in a restaurant. This issue can solve by providing feedback forms in an app.

Low-fidelity prototype

The low-fidelity prototype connected the primary user flow of building and ordering a pizza, so the prototype could be used in a usability study with users.

View the FoodTracky Low-fidility prototype.




High-fidelity prototype

The final highfidelity prototype presented cleaner user flows for building a pizza and checkout. It also met user needs for pickup or delivery options as well as more customization.
